## مشروع التخرج في كلية الحاسبات وتقنية المعلومات (FCIT)

قبل ما ندخل في الأفكار، خلنا نفهم كيف يشتغل نظام مشروع التخرج في FCIT. الكلية فيها ثلاثة أقسام، وكل قسم عنده مادتين للمشروع: الأولى للتخطيط والتصميم، والثانية للتنفيذ والمناقشة.

| | علوم الحاسب (CS) | تقنية المعلومات (IT) | نظم المعلومات (IS) |
|---|---|---|---|
| **رمز المادة** | CPCS 498 / 499 | CPIT 498 / 499 | CPIS 498 / 499 |
| **الساعات** | 1 + 3 | 1 + 3 | 1 + 3 |
| **التركيز** | خوارزميات، ذكاء اصطناعي، أنظمة | شبكات، أمن سيبراني، بنية تحتية | أنظمة أعمال، تحليل بيانات |
| **اللغة الأساسية** | Java | Web Stack (PHP, JS, React) | متنوعة |

**الفصل الأول (498):** تختار الفكرة، تكتب الاقتراح (Proposal)، تسوي دراسة الجدوى، وتصمم النظام (UML diagrams). ساعة معتمدة واحدة بس، لكنها الأساس اللي يبني عليه كل شيء.

**الفصل الثاني (499):** تنفّذ المشروع بالكامل، تختبره، تكتب التقرير النهائي، وتقدمه أمام لجنة المناقشة. ثلاث ساعات معتمدة، وهي المرحلة اللي تحدد تقييمك النهائي.

الكلية حاصلة على اعتماد ABET، وهذا يعني إن لجان المناقشة تتوقع منك منهجية واضحة في التخطيط والتوثيق وليس فقط كود يشتغل.

<Callout type="info" title="ملاحظة عن لغات البرمجة">
في FCIT، جافا هي اللغة الأساسية اللي تتعلمها من CPCS 202 إلى CPCS 204. لكن في مشروع التخرج تقدر تستخدم أي لغة بموافقة المشرف. طلاب IT عندهم خبرة في PHP و React و MySQL من مقرراتهم. لو تبي مقارنة شاملة بين اللغات، اقرأ [دليل اختيار لغة البرمجة لمشروع التخرج](/blog/best-programming-languages-graduation-projects).
</Callout>

## أفكار مشاريع تخرج قسم علوم الحاسب (CPCS 499)

مشاريع قسم علوم الحاسب تركّز عادة على الخوارزميات، الذكاء الاصطناعي، تصميم الأنظمة، أو حل مشاكل حسابية معقدة. أنت كطالب CS درست جافا بعمق من خلال [CPCS 202](/blog/cpcs-202-programming-1-kau) و [CPCS 204](/blog/cpcs-204-data-structures-kau)، وهذا يعطيك أساس قوي تبني عليه.

### 1. نظام تصنيف النصوص العربية بالذكاء الاصطناعي

نظام يحلل النصوص العربية ويصنّفها تلقائيا (إيجابي، سلبي، محايد) باستخدام نماذج معالجة اللغة الطبيعية. مفيد لتحليل تقييمات المنتجات أو آراء الطلاب في الاستبيانات. التحدي الأساسي هو التعامل مع اللهجات العربية المختلفة والعامية السعودية تحديدا.

**التقنيات المقترحة:** Python، AraBERT أو CAMeL Tools، Flask، PostgreSQL

**مستوى الصعوبة:** متقدم

### 2. محرك توصية المقررات الدراسية لطلاب FCIT

نظام يحلل سجل الطالب الأكاديمي ويقترح عليه المقررات المناسبة للفصل القادم بناء على معدله ومتطلباته المتبقية وتقييمات الطلاب السابقين. يستخدم خوارزميات التوصية (Collaborative Filtering) لاقتراح الجداول المثالية.

**التقنيات المقترحة:** Java Spring Boot، MySQL، خوارزميات ML، React

**مستوى الصعوبة:** متقدم

### 3. كاشف الانتحال العلمي للنصوص العربية

أداة تفحص الأبحاث والتقارير العربية وتكتشف النصوص المنسوخة أو المعاد صياغتها بشكل سطحي. يقارن النص المدخل بقاعدة بيانات من الأبحاث المنشورة ويحسب نسبة التشابه باستخدام Cosine Similarity و TF-IDF.

**التقنيات المقترحة:** Python، NLP، Scikit-learn، Flask، PostgreSQL

**مستوى الصعوبة:** متقدم

### 4. تطبيق تعلّم هياكل البيانات بشكل مرئي تفاعلي

تطبيق تعليمي يعرض هياكل البيانات (Linked Lists, Stacks, Queues, Trees) بشكل مرئي متحرك. الطالب يدخل العمليات (إضافة، حذف، بحث) ويشاهد التنفيذ خطوة بخطوة مع تتبع التعقيد الزمني. فكرة ممتازة لأنك تستفيد مباشرة من محتوى CPCS 204.

**التقنيات المقترحة:** React أو JavaFX، Canvas API أو D3.js، TypeScript

**مستوى الصعوبة:** متوسط

### 5. نظام حجز مواعيد العيادات الجامعية

نظام ويب يتيح لطلاب الجامعة حجز مواعيد في العيادات الصحية بالحرم الجامعي. يشمل تسجيل الدخول، عرض الأوقات المتاحة، الحجز، التذكير عبر البريد الإلكتروني، ولوحة تحكم للإدارة.

**التقنيات المقترحة:** Java Spring Boot، React، MySQL، REST API، JavaMail

**مستوى الصعوبة:** متوسط

### 6. روبوت محادثة للإرشاد الأكاديمي في FCIT

شات بوت ذكي يجيب على أسئلة الطلاب حول الخطة الدراسية والمقررات والمتطلبات السابقة في FCIT. يتم تدريبه على دليل الكلية والكتالوج الأكاديمي، ويستطيع الإجابة بالعربية على أسئلة مثل "وش المتطلب السابق لمادة CPCS 204؟" أو "كم ساعة باقي لي للتخرج؟"

**التقنيات المقترحة:** Python، LangChain، OpenAI API، FastAPI، React

**مستوى الصعوبة:** متقدم

### 7. مراقب أداء الشبكات بتعلم الآلة

أداة تراقب حركة البيانات في الشبكة ويكتشف الأنماط غير الطبيعية (Anomaly Detection) اللي ممكن تدل على هجمات أو أعطال. يستخدم خوارزميات تعلم الآلة للتمييز بين الحركة الطبيعية والمشبوهة.

**التقنيات المقترحة:** Python، Scikit-learn، Pandas، Streamlit، Wireshark

**مستوى الصعوبة:** متقدم

### 8. تطبيق موبايل لإدارة المشاريع البرمجية للفرق الصغيرة

تطبيق يساعد فرق التطوير الصغيرة (3-5 أشخاص) على تنظيم مهامهم، تتبع التقدم، ومشاركة الملاحظات. يشمل لوحة كانبان (Kanban Board)، إشعارات فورية، وربط مع GitHub لعرض آخر التعديلات.

**التقنيات المقترحة:** Flutter أو React Native، Firebase، GitHub API، Push Notifications

**مستوى الصعوبة:** متوسط

## أفكار مشاريع تخرج قسم تقنية المعلومات (CPIT 499)

مشاريع قسم تقنية المعلومات تركّز على البنية التحتية، الشبكات، الأمن السيبراني، وتطبيقات الويب العملية. طلاب IT في FCIT عندهم خبرة في تطوير الويب (HTML, CSS, JavaScript, React, PHP, MySQL) من خلال مقررات مثل CPIT 405.

### 1. منصة إدارة التذاكر التقنية لمكتب الدعم الفني

نظام ويب يتيح للموظفين في المؤسسة رفع طلبات الدعم التقني، وللفريق التقني إدارة هذه الطلبات وتصنيفها وتتبع حالتها. يشمل أولويات (عاجل، عادي، منخفض)، إشعارات بريدية، وتقارير إحصائية شهرية.

**التقنيات المقترحة:** PHP Laravel، MySQL، Bootstrap أو Tailwind، REST API

**مستوى الصعوبة:** متوسط

### 2. لوحة مراقبة البنية التحتية للشبكة

لوحة تحكم تعرض حالة الأجهزة والسيرفرات على الشبكة في الوقت الفعلي. تكتشف الأعطال تلقائيا وترسل تنبيهات للمسؤولين. تعرض رسوم بيانية لاستهلاك الموارد (CPU, RAM, Bandwidth) عبر الزمن.

**التقنيات المقترحة:** Python، SNMP Protocol، React، Chart.js، PostgreSQL

**مستوى الصعوبة:** متوسط الى متقدم

### 3. منصة تعليمية تفاعلية لأساسيات الأمن السيبراني

منصة تعليمية تتضمن دروسا نظرية ومحاكاة عملية لهجمات شائعة (SQL Injection, XSS, Phishing) في بيئة آمنة ومعزولة. الطالب يتعلم الهجمة ويجرّبها ويتعلم كيف يحمي منها.

**التقنيات المقترحة:** React، Node.js، Docker للمحاكاة، MySQL

**مستوى الصعوبة:** متوسط الى متقدم

### 4. نظام إدارة الأصول التقنية للمؤسسات

نظام لتتبع جميع الأجهزة والمعدات التقنية في المؤسسة (أجهزة حاسب، طابعات، سيرفرات). يسجل بيانات كل جهاز، تاريخ الشراء، حالة الصيانة، والموظف المسؤول عنه. يدعم مسح الباركود لتسهيل الجرد.

**التقنيات المقترحة:** Laravel، MySQL، Barcode Scanner API، REST API

**مستوى الصعوبة:** متوسط

### 5. موقع تقييم ومراجعة المقررات الجامعية

منصة تتيح لطلاب الجامعة تقييم المقررات اللي درسوها بشكل مجهول. يقيّمون صعوبة المادة، جودة المحتوى، وطريقة التدريس. الطلاب الجدد يستفيدون من هذه التقييمات عند تسجيل موادهم.

**التقنيات المقترحة:** React، Node.js Express، MongoDB، JWT Authentication

**مستوى الصعوبة:** متوسط

### 6. أداة النسخ الاحتياطي السحابي التلقائي

أداة تسوي نسخ احتياطي تلقائي مجدول للملفات والمجلدات المحددة إلى خدمة سحابية. يدعم جدولة مرنة (يومي، أسبوعي)، ضغط الملفات، تشفير البيانات قبل الرفع، وإشعارات بنجاح أو فشل العملية.

**التقنيات المقترحة:** Python، AWS S3 أو Google Cloud Storage، Flask، Cron Jobs

**مستوى الصعوبة:** متوسط

### 7. تطبيق ويب لإدارة الفعاليات والأندية الطلابية

منصة تسمح للأندية الطلابية في الجامعة بإنشاء فعالياتهم وإدارة التسجيل وإصدار شهادات الحضور تلقائيا. تشمل لوحة تحكم لرئيس النادي وصفحة عامة لعرض الفعاليات القادمة.

**التقنيات المقترحة:** React أو Next.js، Supabase أو Firebase، Tailwind CSS

**مستوى الصعوبة:** متوسط

### 8. كاشف التهديدات الأمنية من تحليل السجلات

أداة تجمع سجلات الأحداث (Logs) من الأنظمة والسيرفرات، يحللها باستخدام قواعد محددة وخوارزميات بسيطة، ويكتشف الأنشطة المشبوهة مثل محاولات الدخول المتكررة الفاشلة أو الوصول لملفات غير مصرح بها.

**التقنيات المقترحة:** Python، ELK Stack (Elasticsearch, Logstash, Kibana)، Flask

**مستوى الصعوبة:** متقدم

## أفكار مشاريع تخرج قسم نظم المعلومات (CPIS 499)

مشاريع قسم نظم المعلومات تركّز على حل مشاكل تنظيمية وإدارية باستخدام التقنية. الفكرة في مشاريع IS إنك تبني نظام يحسّن عملية أعمال موجودة أو يحوّل عملية يدوية إلى رقمية. لجنة المناقشة تهتم بالقيمة اللي يقدمها النظام للمستخدم النهائي بقدر اهتمامها بالجانب التقني.

### 1. نظام إدارة علاقات العملاء (CRM) للمنشآت الصغيرة

نظام يساعد المنشآت الصغيرة على تتبع عملائها، إدارة التواصل معهم، وتسجيل فرص المبيعات. يشمل سجل تفاعلات مع كل عميل، تذكيرات بالمتابعة، وتقارير أداء المبيعات.

**التقنيات المقترحة:** Laravel أو Django، MySQL، Chart.js، REST API

**مستوى الصعوبة:** متوسط

### 2. داشبورد تحليلي لبيانات المبيعات

لوحة تحكم تفاعلية تعرض بيانات المبيعات بطرق مرئية (رسوم بيانية، خرائط حرارية، مؤشرات أداء). تسمح بالفلترة حسب الفترة الزمنية والمنتج والمنطقة. تساعد المدراء على اتخاذ قرارات مبنية على البيانات.

**التقنيات المقترحة:** Python، Pandas، Plotly أو Dash، PostgreSQL

**مستوى الصعوبة:** متوسط

### 3. نظام إدارة المخزون والمستودعات

نظام يتتبع حركة البضائع في المستودع (دخول، خروج، نقل). يرسل تنبيهات عند وصول المخزون لحد أدنى محدد، ويولّد تقارير جرد دورية. يدعم مسح الباركود لتسريع العمليات.

**التقنيات المقترحة:** React، Spring Boot أو Laravel، MySQL، Barcode API

**مستوى الصعوبة:** متوسط

### 4. منصة إلكترونية لإدارة عمليات التوظيف

نظام يدير دورة التوظيف كاملة: من نشر الوظيفة، استقبال طلبات المتقدمين، فرزها، جدولة المقابلات، حتى إصدار العرض الوظيفي. يوفّر لوحة تحكم لقسم الموارد البشرية ولوحة منفصلة للمتقدمين لمتابعة حالة طلباتهم.

**التقنيات المقترحة:** Laravel، Vue.js أو React، MySQL، REST API

**مستوى الصعوبة:** متوسط

### 5. نظام تقييم أداء الموظفين

نظام يتيح للمدراء تقييم موظفيهم بشكل دوري وفق مؤشرات أداء محددة (KPIs). يتتبع التقييمات عبر الزمن، يولّد تقارير أداء فردية وجماعية، ويدعم التقييم 360 درجة (تقييم من الزملاء والمدير).

**التقنيات المقترحة:** React، Node.js Express، MongoDB، Chart.js

**مستوى الصعوبة:** متوسط

### 6. نظام إدارة المشتريات والموردين

نظام يدير عملية الشراء من إنشاء طلب الشراء، مقارنة عروض الأسعار، إصدار أمر الشراء، استلام البضاعة، حتى الدفع. يتتبع الموردين وتقييمهم ويولّد تقارير المصاريف.

**التقنيات المقترحة:** Django، PostgreSQL، REST API، مكتبة توليد PDF

**مستوى الصعوبة:** متوسط الى متقدم

### 7. تطبيق ويب لإدارة العقود والاتفاقيات

نظام لتسجيل العقود وتتبع مواعيد انتهائها والتجديد. يرسل تنبيهات قبل انتهاء العقد بفترة محددة، يحفظ نسخ رقمية من المستندات، ويوفّر بحثا متقدما حسب نوع العقد أو الطرف الآخر.

**التقنيات المقترحة:** Laravel، MySQL، PDF.js، Digital Signature API

**مستوى الصعوبة:** متوسط

### 8. نظام تحليل استبيانات رضا العملاء

نظام يتيح إنشاء استبيانات مخصصة، توزيعها على العملاء، جمع الردود، وتحليلها تلقائيا. يستخدم تحليل المشاعر (Sentiment Analysis) على الإجابات النصية المفتوحة ويعرض النتائج في لوحة تحكم تفاعلية.

**التقنيات المقترحة:** Python، Flask، Pandas، NLP، React

**مستوى الصعوبة:** متوسط الى متقدم

<Callout type="tip" title="نصيحة لطلاب IS">
في مناقشة مشروع التخرج، لجنة نظم المعلومات تركّز كثير على **القيمة اللي يضيفها النظام للمستخدم**. لا تكتفي بشرح الكود، بل اشرح كيف النظام يحل مشكلة حقيقية ويوفّر وقت أو جهد أو فلوس. لو تبي نصائح تفصيلية للمناقشة، اقرأ [دليل تقديم مشروع التخرج أمام اللجنة](/blog/graduation-project-presentation-guide).
</Callout>

## نصائح عملية لمشروع التخرج في FCIT

### الفصل الأول (498): أسّس مشروعك صح

<StepList title="خطوات الفصل الأول">
1. **اختر فكرتك في أول أسبوعين:** لا تأجل. اختر 2-3 أفكار من القائمة فوق، ناقشها مع مشرفك، واحسم قرارك بسرعة
2. **اكتب وصف المشكلة بوضوح:** المشرف واللجنة يريدون يعرفون وش المشكلة اللي تحلها بالضبط. جملة واحدة واضحة أفضل من صفحة غامضة
3. **سوّ دراسة الجدوى:** قارن حلك بالحلول الموجودة، ووضّح ليش حلك مختلف أو أفضل
4. **ارسم مخططات UML:** Use Case Diagram و Class Diagram و Sequence Diagram على الأقل. هذي من متطلبات ABET
5. **ابنِ نموذج أولي (Prototype):** حتى لو بسيط، نموذج شغّال يبيّن إنك فاهم المطلوب ويقنع المشرف إن المشروع واقعي
</StepList>

لو تبي تتعمق في كيفية كتابة خطة المشروع بالتفصيل، اقرأ [دليل كتابة خطة مشروع التخرج خطوة بخطوة](/blog/how-to-write-graduation-project-plan).

### الفصل الثاني (499): نفّذ وجهّز للمناقشة

<StepList title="خطوات الفصل الثاني">
1. **ابدأ البرمجة من أول أسبوع:** لا تضيع وقت في إعادة التخطيط. الخطة جاهزة من 498، ابدأ التنفيذ فورا
2. **استخدم Git من اليوم الأول:** المشرف يقدر يتابع تقدمك، وأنت تحمي شغلك من الضياع. [راجع دليل Git للطلاب](/blog/git-version-control-guide)
3. **اختبر بشكل مستمر:** لا تخلي التيست لآخر أسبوع. اختبر كل وظيفة أول ما تخلصها
4. **اكتب التقرير بالتوازي:** لا تنتظر تخلص الكود عشان تبدأ التقرير. اكتب كل فصل بعد ما تنتهي من الجزء المقابل له. [دليل كتابة تقرير المشروع](/blog/graduation-project-report-guide)
5. **تدرّب على العرض 3 مرات على الأقل:** سجّل نفسك بالفيديو وشاهد الأخطاء. [نصائح تقديم العرض أمام اللجنة](/blog/graduation-project-presentation-guide)
</StepList>

<Callout type="warning" title="خطأ شائع عند طلاب FCIT">
أكبر خطأ يرتكبه الطلاب هو قضاء فصل 498 بالكامل في التردد بين الأفكار بدون ما يحسمون قرارهم. نهاية الفصل تلاقيهم بدون اقتراح مكتمل ولا تصميم واضح. النتيجة؟ يدخلون 499 وهم لسه يخططون بدل ما ينفذون. اختر فكرتك بسرعة، حتى لو مو مثالية، فكرة متوسطة منفّذة صح أفضل بكثير من فكرة عبقرية ما اكتملت.
</Callout>

### اختيار الفريق والتقنيات

لو المشروع جماعي، اختر فريقك بعناية. الوضع المثالي يكون عندك شخص قوي في الواجهة الأمامية (Frontend)، شخص في الباك إند (Backend)، وشخص يتولى قاعدة البيانات والاختبار.

بالنسبة للتقنيات، هذي أكثر التركيبات المجربة عند طلاب FCIT:

- **لمشاريع CS:** Java Spring Boot + React + MySQL (تستفيد من خبرتك في جافا)
- **لمشاريع IT:** PHP Laravel + MySQL + Tailwind CSS أو React + Node.js + MongoDB (تستفيد من مقررات الويب)
- **لمشاريع IS:** Django أو Laravel + PostgreSQL + Chart.js (مناسب لأنظمة الأعمال ولوحات التحكم)
- **لمشاريع الذكاء الاصطناعي:** Python + Flask/FastAPI + Scikit-learn أو TensorFlow + React (للواجهة)

## كيف تختار الفكرة المناسبة من القائمة

الحين عندك 24 فكرة. لا تحاول تختار "الأفضل" لأن ما في فكرة أفضل بشكل مطلق. الفكرة المناسبة هي اللي تتقاطع فيها ثلاثة أشياء: مهاراتك الحالية، اهتمامك الشخصي، والوقت المتاح عندك.

ابدأ بحذف كل فكرة مستوى صعوبتها أعلى من قدراتك الحالية. ثم من اللي بقوا، اختر 2-3 أفكار تحمسك فعلا. ناقشها مع مشرفك واحسم قرارك. لو تبي منهجية مفصلة أكثر، اقرأ [دليل اختيار فكرة مشروع التخرج](/blog/how-to-choose-graduation-project-idea).

بالتوفيق في مشروع تخرجك. وتذكّر: كل طالب تخرّج قبلك مر بنفس القلق اللي تمر فيه الحين، والنتيجة كانت إنهم أنجزوا مشاريعهم ونجحوا. أنت تقدر تسويها.

## أسئلة شائعة عن مشروع التخرج في FCIT

### كم مدة مشروع التخرج في كلية الحاسبات بجامعة الملك عبدالعزيز؟

مشروع التخرج يمتد على فصلين دراسيين. الفصل الأول (498) مخصص للتخطيط وكتابة الاقتراح وتصميم النظام (ساعة معتمدة واحدة). الفصل الثاني (499) مخصص للتنفيذ والاختبار وكتابة التقرير والمناقشة (3 ساعات معتمدة).

### هل أقدر أستخدم لغة برمجة غير جافا في مشروع التخرج؟

نعم. رغم أن جافا هي اللغة الأساسية في المنهج، تقدر تستخدم أي لغة (Python، JavaScript، PHP، وغيرها) بشرط موافقة المشرف. الأهم أن اللغة تكون مناسبة لطبيعة مشروعك.

### هل مشروع التخرج فردي أو جماعي؟

يعتمد على القسم والمشرف. بعض المشاريع تكون فردية وبعضها جماعية (2-3 طلاب عادة). المشاريع الجماعية تحتاج توزيع مهام واضح لأن اللجنة تقيّم كل طالب على حدة.

### وش أسوي لو المشرف رفض فكرتي؟

لا تأخذها بشكل شخصي. اسأل المشرف عن سبب الرفض بالتحديد: هل الفكرة مكررة؟ كبيرة جدا؟ خارج تخصص القسم؟ بناء على السبب، عدّل الفكرة أو اختر فكرة ثانية من القائمة وارجع له بسرعة. لا تضيع أسابيع في الإقناع.
